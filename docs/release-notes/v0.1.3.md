# Clawdeck v0.1.3

This is a first-run polish release after live npm-user testing.

## Added

- `clawdeck setup`: a one-command first-run path that adopts the workspace, previews or applies the backed-up local profile, runs the readiness drill, prints model pull commands, and leaves one next action.

## Fixes

- Boolean flags no longer consume the next positional argument, so commands like `clawdeck local --force ./workspace` create the requested workspace instead of writing into the current directory.
- Value flags now fail clearly when the value is missing, for example `clawdeck apply --home`.
- `clawdeck snapshot` now accepts `--home dir`, matching the existing snapshot API and error guidance.
- Unknown commands and invalid options now fail before doing any workspace work.
- `clawdeck audit` now follows the configured OpenClaw workspace when the current directory is not itself a Clawdeck/OpenClaw workspace.
- `clawdeck <command> --help` now prints help instead of treating `--help` as an invalid subcommand option.

## Verification

- Source test suite: 22/22 passing.
- Published-package user path retested through clean npm install and `npx`.
- Local package path retested against fresh temp workspaces, paths with spaces, missing-value flags, snapshot redaction, setup/adopt/apply/drill/audit/handoff, and smoke failure behavior.
